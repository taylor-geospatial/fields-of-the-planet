# Remote Sensing — Spectral Complexity

<Columns cols={2} align="center">
<div>

In computer vision, images are RGB. In remote sensing, sensors vary dramatically:

- **Spectral bands** — Landsat 8: 11 bands, Hyperion: 242 bands
- **Wavelengths** — optical: 0.4–15 µm, SAR: 7.5 mm–1 m
- **Spatial resolution** — GOES: 4 km/px → drone: 7 mm/px
- **Temporal frequency** — continuous to annual

# Geospatial Sampling

<Columns cols={2} align="center">
<div>

**TorchVision datasets** are simple:
- Fixed length, indexed by integer

**Geospatial datasets** are harder:
- Raster scenes are too large to load at once
- Vector files have no pixels, don't align with rasters

**Solution:** index by **(x₁, x₂, y₁, y₂, t₁, t₂)** spatiotemporal bounding box

# Abstract Base Classes

<Columns cols={2} align="start">
<div>

### GeoDataset
- Backed by an **R-tree** for spatiotemporal indexing
- Supports spatial + temporal queries
- `__and__` → IntersectionDataset
- `__or__` → UnionDataset

### NonGeoDataset
- Standard integer-indexed dataset
- Used for curated benchmark datasets
- Compatible with `torch.utils.data.DataLoader`

</div>
<div>

```python
# GeoDataset — spatiotemporal R-tree index
landsat = Landsat8(paths="...")
cdl     = CDL(paths="...")
dataset = landsat & cdl  # IntersectionDataset
dataset = landsat | cdl  # UnionDataset

# NonGeoDataset — integer index
dataset = RESISC45(root="...", split="train")
sample  = dataset[42]
# {'image': Tensor[C,H,W], 'label': int}
```

</div>
</Columns>

# Combining Datasets — Code

```python
from torch.utils.data import DataLoader
from torchgeo.datasets import CDL, Landsat7, Landsat8, stack_samples
from torchgeo.samplers import RandomGeoSampler

# v0.9+: band names use full filename prefix (SR_B*)
landsat7 = Landsat7(paths="...", bands=["SR_B1","SR_B2","SR_B3","SR_B4","SR_B5","SR_B7"])
landsat8 = Landsat8(paths="...", bands=["SR_B2","SR_B3","SR_B4","SR_B5","SR_B6","SR_B7"])
cdl      = CDL(paths="...")

dataset  = (landsat7 | landsat8) & cdl  # union then intersection

sampler  = RandomGeoSampler(dataset, size=256, length=10_000)
loader   = DataLoader(dataset, batch_size=128, sampler=sampler, collate_fn=stack_samples)

for batch in loader:
    ...  # image: Tensor[B,C,H,W]   label: Tensor[B,H,W]
```

# Pre-trained Models

Each satellite has a **unique set of spectral bands**. TorchGeo ships the first library of models pre-trained on **multispectral imagery**.

```python
from torchgeo.models import resnet50, ResNet50_Weights, unet, Unet_Weights

# ResNet50 pre-trained on all Sentinel-2 bands via MoCo
model = resnet50(weights=ResNet50_Weights.SENTINEL2_ALL_MOCO)

# U-Net pre-trained for 3-class field boundary segmentation (FTW)
model = unet(weights=Unet_Weights.SENTINEL2_3CLASS_FTW)
```

**Available backbones:** ResNet · ViT · Swin · Change Star · CROMA · DoFA · FarSeg · Scale-MAE · Copernicus-FM · Panopticon · OlmoEarth · Tessera · Presto · and more

---

# Lightning Integration — Trainers

`DataModules` and `Trainers` make it trivially easy to train and evaluate models.

```python
from lightning.pytorch import Trainer
from torchgeo.datamodules import TropicalCycloneDataModule
from torchgeo.trainers import RegressionTask

datamodule = TropicalCycloneDataModule(root="...", batch_size=64)
model = RegressionTask(model="resnet18", learning_rate=0.1)
trainer = Trainer(min_epochs=20, max_epochs=100)

trainer.fit(model=model, datamodule=datamodule)
trainer.test(model=model, datamodule=datamodule)
```

Supports classification, regression, semantic segmentation, change detection, and instance segmentation out of the box.

A **foundation model** reads a satellite image and outputs a compact list of numbers — a fingerprint that captures what that place looks like.

</div>

# The Storage Problem

Running a model over the whole Earth at 10 m resolution produces a staggering amount of embedding data.

<Columns cols={2} align="center">
<div>

Think of it like this: if the satellite imagery is a **filing cabinet**, the embeddings are a **warehouse**.

- 1.5 trillion image patches per year (global 10 m)
- Each embedding: 1,024 numbers × 4 bytes = **4 KB per patch**
- Total: **6 petabytes — $1.7M/yr just in cloud storage**

> The fingerprints cost more to store than the photos.

</div>
<div>

# Why Does Compression Work So Well?

Embeddings look like 1,024 numbers, but most of that is **redundant**.

<Columns cols={2} align="center">
<div>

**Analogy: JPEG compression**

A raw photo is millions of pixels — but neighboring pixels are correlated. JPEG exploits that redundancy and throws away what your eye won't miss. Same idea here.

<br/>

**Earth embeddings are the same:**
- DINOv3 has 1,024 dimensions
- But only **13–17 dimensions carry real information**
- The rest is noise

</div>
<div>

<div style={{ textAlign: 'center' }}>
<div style={{ fontSize: '3.5rem', fontWeight: 'bold', color: 'var(--theme-ui-colors-primary)', lineHeight: 1.1 }}>1,024</div>
<div style={{ fontSize: '1em', color: 'var(--theme-ui-colors-textMuted)', marginBottom: '1.2rem' }}>stored dimensions</div>

# What is Retrieval?

**Given one satellite image, find all similar-looking places across the planet.**

<Columns cols={2} align="center">
<div>

Think of it like **reverse image search** — but for the Earth.

1. Draw a region (e.g., agricultural fields in Iowa)
2. Compute its embedding fingerprint
3. Compare against 50 million other fingerprints
4. Return the most similar locations — anywhere on Earth

**Why is this useful?**
- Find crop fields that look like yours in other countries
- Spot deforestation patterns matching known examples
- Identify similar terrain for military or disaster response

</div>

# TerraBit — Planetary Retrieval in the Browser

**49.8M embeddings over CONUS. No server. Runs entirely in your browser.**

<Columns cols={2} align="center">
<div>

**The data:**
- Clay v1.5 foundation model embeddings
- Compressed from **183 GB → 11 GB** (16× smaller) using binary quantization

**How it works:**
- Draw a region on the map
- Your browser computes the similarity against all 49.8M tiles
- Returns ranked results in under a second

</div>
<div>

**What you can do:**
- Rectangle or freehand polygon queries
- Combine multiple examples with **AND / OR / NOT**
- Export results as **GeoParquet**
- No cloud infrastructure needed

</div>
